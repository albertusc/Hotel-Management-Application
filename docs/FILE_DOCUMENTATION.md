# File Documentation Index

This document describes the role of important files in the repository.

## Root-Level Files

- `WindowsFormsApp1.sln`: Solution entry that loads the Windows Forms project.
- `README.md`: Main project overview, setup, screenshots, and privacy notes.
- `.gitignore`: Excludes machine-local and build artifacts.

## SQL Folder

- `SQL/VisProject.sql`: Full database DDL + seed data script.
- `SQL/README.md`: Data model and table-by-table SQL documentation.

## Images Folder

Contains UI screenshots and ER diagram used for documentation and presentation.

## Main Project: WindowsFormsApp1

### Core App Files

- `WindowsFormsApp1/Program.cs`: Application entry point.
- `WindowsFormsApp1/App.config`: Runtime config and connection strings.
- `WindowsFormsApp1/Form1.cs`: Login/authentication form logic.
- `WindowsFormsApp1/Form2.cs`: Main navigation form after login.
- `WindowsFormsApp1/cpesan.cs`: Project-specific helper/model class (kept as source).

### Form Designer/Resources

For each form class, Visual Studio also maintains:

- `*.Designer.cs`: Auto-generated UI layout/control initialization.
- `*.resx`: Form resource file.

### Administrasi Module

- `WindowsFormsApp1/Administrasi/PengDK.cs`: Employee data management CRUD logic.
- `WindowsFormsApp1/Administrasi/SisPeng.cs`: Payroll retrieval, calculation, and txt export/read logic.

### Inventory Module

- `WindowsFormsApp1/Inventory/InvBrng.cs`: Inventory item CRUD and data loading.
- `WindowsFormsApp1/Inventory/PemStok.cs`: Stock maintenance and related operations.

### Housekeeping Module

- `WindowsFormsApp1/Housekeeping/InptDatBrngKeber.cs`: Housekeeping item input/maintenance.
- `WindowsFormsApp1/Housekeeping/KetersediaanKebersihan.cs`: Housekeeping stock availability view/logic.

### Resepsionis Module

- `WindowsFormsApp1/Resepsionis/GuestR.cs`: Guest reception flow.
- `WindowsFormsApp1/Resepsionis/GuestPay.cs`: Payment flow.
- `WindowsFormsApp1/Resepsionis/GuestCheckout.cs`: Checkout flow.
- `WindowsFormsApp1/Resepsionis/RoomReserv.cs`: Room reservation logic.
- `WindowsFormsApp1/Resepsionis/FormDataReserv.cs`: Reservation report form.
- `WindowsFormsApp1/Resepsionis/FormDataReservUnsort.cs`: Alternate reservation listing form.
- `WindowsFormsApp1/Resepsionis/CrystalReport1.cs`, `DataReserv.cs`, `DataReservUnsort.cs`: Report class wrappers.
- `WindowsFormsApp1/Resepsionis/*.rpt`: Crystal Reports templates.

### Typed DataSets and Settings

- `WindowsFormsApp1/VisProjectDataSet*.xsd`: Dataset schema definitions per module.
- `WindowsFormsApp1/VisProjectDataSet*.Designer.cs`: Generated dataset/table-adapter code.
- `WindowsFormsApp1/VisProjectDataSet*.xsc`, `*.xss`: Dataset support metadata files.
- `WindowsFormsApp1/Properties/Settings.settings`: Application setting source.
- `WindowsFormsApp1/Properties/Settings.Designer.cs`: Generated settings accessor class.
- `WindowsFormsApp1/Properties/AssemblyInfo.cs`: Assembly metadata.
- `WindowsFormsApp1/Properties/Resources.resx`, `Resources.Designer.cs`: Embedded resource definitions/accessor.

### Text Data

- `WindowsFormsApp1/Txt/DataGaji.txt`: Payroll text storage file consumed by `SisPeng.cs`.

## Notes on Generated Files

- `*.Designer.cs`, dataset designer files, and settings designer files are generated by Visual Studio tools.
- Edits to generated files should be minimized and usually performed through the associated source designer file (`.resx`, `.settings`, `.xsd`) to avoid regeneration conflicts.
